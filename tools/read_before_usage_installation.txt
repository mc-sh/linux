POUR UTILISATION (CHIPSEC)
apres source venv
PERMET DE VOIR LES MODULES INSTALLÉ
sudo python3 -m chipsec_main -i
UTILISER UN MODULE
sudo python3 -m chipsec_main -m common.spi_access -v

MODULE À UTILISÉ
common.spi_access -v
common.spi_lock -v
common.spi_desc -v (CRITIQUE module important pour fd (flash descriptor (permmission écriture))
common.spi_fdopss -v (CRITIQUE fd bypasser d'usine, regarde si présent)
common.bios_wp -v BONUS --> sudo flashrom -p internal --wp-status (flashrom prend en consideration les protection logiciel exemple si iomem=relaxed n'est pas activé
common.me_mfg_mode -v
common.bios_kbrd_buffer -v (moins important, rarement un vecteur sur carte moderne)
common.smrr -v
common.memconfig -v
common.bios_smi -v
common.ia32cfg -v
common.smm -v
common.uefi.access_uefispec -v
common.uefi.s3bootscript -v (moins important, possiblement plus supporté)
common.secureboot.variables -v
common.smm_dma -v

CONFIRMATION HORS DE TOUT DOUTE SI
toujours faire les test en mode UEFI pour une bonne lecture
fd est fermé
me pas en manufacturing mode
bios wp/smm activé
secure boot bien configuré

POUR VERIFIER AVANT UTILISÉ FLASHROM ET CHIPSEC
cat /sys/kernel/security/lockdown 
si pas [none]
